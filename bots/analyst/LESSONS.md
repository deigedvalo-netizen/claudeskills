## Candidates (bot-proposed; a human moves lines up to Approved)
- When a repo already ships a "v2" visual system inline, distinguish what physically exists (a single stylesheet) from what a milestone names as a separate artifact (token file, catalog, checklist) and mark the latter greenfield explicitly.
- Pin the backend contract by enumerating the exact route table and the response fields the frontend reads, so an "unchanged backend" acceptance criterion has concrete anchors.
- For a no-build static frontend, confirm the absence of tooling by checking `package.json` scripts, config dotfiles, and `node_modules` rather than assuming.
